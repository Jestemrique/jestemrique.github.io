The reason for creating a static blog template is to have a quick and easy method of putting together a website in a matter of minutes. I used this template as a starting potin to build this blog. I also use it to spin up a tet environment whenever I need to test something locally.

The template can be found in the Github repository jekyll-uikit-template

How to install and use the template:

Note: As requirements, Docker and Nodejs need to be installed on our computer to create Jekyll and UIKit service.

First of all we need to clone or download the repository to our machine.

git clone https://github.com/Jestemrique/jekyll-uikit-template.git

Once the repository is cloned, we can rename the directory to any other name

mv jekyll-uikit-template my-fav-name

and remove the hidden directories from git (.git) to create a new repository from scratch.

Once the above is done, execute `docker-compose up -d` so that Docker does all the work and have the site ready to be seen in our browser through port 4000

localhost: 4000

Explanation of docker-compose.yml

There are two services in docker-compose.yml:

  • jekyll: In charge of installing the Jekyll image and launching the site. For this service, Jekyll has an image on **Docker Hub** that can be downloaded from [jekyl/jekyll](https://hub.docker.com/r/jekyll/jekyll). More detailed information can be found in their Github repo [engygeeks/jekyll-docker](https://github.com/envygeeks/jekyll-docker)
  • uikit: To have UIKit available, an image of **Node.js** `node:latest` will help us to install the UIKit modules specified in `package.json` 
  File: package.son
  
  {
    "name": "uikit",
    "version": "1.0.0",
    "description": "",
    "main": "index.js",
    "scripts": {
      "css-watch": "node-sass _sass/main.scss assets/css/styles.css -w",
      "start": "npm run css-watch",
      "test": "echo \"This is a test.\""
    },
    "keywords": [],
    "author": "",
    "license": "ISC",
    "dependencies": {
      "purecss": "^2.0.5",
      "uikit": "^3.6.20"
    }
  }
