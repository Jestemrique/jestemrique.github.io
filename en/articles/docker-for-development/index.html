Docker for development

May 2, 2026

Tools Development

One of the first decisions in this project was to use Docker instead of installing Node.js directly on my machine.

The problem it solves

Without Docker, I'd have to install Node.js, the right version, and the project tools on my computer. Over time that becomes a mess that's hard to keep clean.

Docker puts all of that into an isolated container: the project carries its own environment and my machine stays clean.

How I start it

To bring the site up I use a single command:

docker compose up --build -d

And to stop it:

docker compose down

What I learned

  • Only the src/ folder is mounted as a volume, so changes there reload on their own.
  • If I change the configuration (.eleventy.js, Dockerfile), I have to rebuild the image with --build.

Having an isolated environment gives me peace of mind: I can break things inside the container without affecting the rest of my system.
