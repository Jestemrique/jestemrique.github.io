Docker para desarrollo

2 de mayo de 2026

Una de las primeras decisiones del proyecto fue usar Docker en lugar de instalar Node.js directamente en mi máquina.

El problema que resuelve

Sin Docker, tendría que instalar Node.js, la versión correcta, y las herramientas del proyecto en mi ordenador. Con el tiempo eso se vuelve un lío difícil de mantener limpio.

Docker mete todo eso en un contenedor aislado: el proyecto trae su propio entorno y mi máquina queda limpia.

Cómo lo arranco

Para levantar el sitio uso un único comando:

docker compose up --build -d

Y para detenerlo:

docker compose down

Lo que aprendí

  • Solo la carpeta src/ está montada como volumen, así que los cambios ahí se recargan solos.
  • Si cambio la configuración (.eleventy.js, Dockerfile), hay que reconstruir la imagen con --build.

Tener el entorno aislado me da tranquilidad: puedo romper cosas dentro del contenedor sin afectar al resto del sistema.
